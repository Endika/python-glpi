Class GLPI

source code

This is a python interface to the GLPI webservices plugin It depends heavily on JSON, urllib and urllib2. Requires python 2.7. Also, has only been tested with v.1.3.0 of the GLPI webservices plugin.

All types are run through urllib2.urlencoder, so the get coerced into strings, but the types the REST API expects are delineated anyway.

This documentation, with few ammendments, has been lifted (without permission) almost ver batim from https://forge.indepnet.net/projects/webservices/wiki/En_devguide.

Thanks so much to all the awesome developers that have made the webservices plugin and GLPI so useful.

Special Note If any boolean arguments are defined, they're automatically added to the GET request, which means the webservices API will treat them as being true. You've been warned.

list_tickets(self, mine=None, user=None, recipient=None, group=None, mygroups=None, category=None, status=None, startdate=None, enddate=None, itemtype=None, item=None, entity=None, satisfaction=None, approval=None, approver=None, order=None, id2name=None, count=None, _help=None)

source code 

Return a JSON serialized list of tickets from the GLPI server.

Parameters:
  • mine (boolean) - list tickets of user or recipient
  • user (integer) - ID of the victim: person concerned by the ticket
  • recipient (integer) - ID of the requestor: person who created the ticket
  • group (integer) - ID of the requestor group (must have show_all_ticket or show_group_ticket right)
  • mygroups (boolean) - list of groups of current user
  • category (integer) - ID of the category of the ticket
  • status (string) - status of the ticket, must be:
    • notold
    • old
    • process
    • waiting
    • old_done
    • new
    • old_notdone
    • assign
    • plan
    • all
  • startdate (datetime) - start of the period where the ticket is active (close date after this)
  • enddate (datetime) - end of the period where the ticket is active (open date before this)
  • itemtype (string) - ID of the type of the item (see GLPI.list_helpdesk_types)
  • item (integer) - ID of the item, requires itemtype
  • entity (integer) - ID of the entity of the ticket
  • satisfaction (integer) - only closed tickets with a satisfaction survey:
    1. all
    2. waiting
    3. answered
  • approval (string) - only tickets with a validation request:
    • all
    • waiting
    • accepted
    • pending
  • approver (integer) - user ID, only tickets with a validation request sent to this user
  • order (list) - list of allowed key names:
    • id
    • date
    • closedate
    • date_mod
    • status
    • users_id
    • groups_id
    • entities_id
    • priority
  • id2name (boolean) - option to enable id to name translation of dropdown menus
  • count (list) - interable including start and limit integers
  • _help (boolean) - get usage information

create_ticket(self, title, content, entity=None, user=None, group=None, requester=None, victim=None, observer=None, date=None, itemtype=None, item=None, urgency=None, _type=None, source=None, category=None, user_email=None, user_email_notification=None, _help=None)

source code 

Returns a JSON serialized version of a ticket upon success. Unless otherwise noted, all params are optional

Parameters:
  • entity (integer) - ID, optional, default is current on, must be in the active ones
  • user (integer or list of integers) - requester ID, default to current logged in user
  • group (integer or list of integers) - requester ID, default is None
  • requester (integer or list of integers) - additional requester(s) with mail notification
  • victim (integer or list of integers) - same as requester, without mail notification
  • observer (integer or list of integers) - additional observers
  • date (ISO formatted date string) - defaults to system date
  • itemtype (integer) - id of the type of the item, optional, default none
  • item (integer) - ID of the item, optional, default none
  • title (string) - required short description of the issue
  • content (string) - required longer description of the issue
  • urgency (integer) - defaults to 3, 1-5 are accepted
  • _type (integer) - type of ticket, defaults to 1
  • source (string) - name of the RequestType, optional, defaults WebServices
  • category (integer) - default is none
  • user_email (string) - enable notification to this email address
  • user_email_notification (boolean) - enable notification to the users email (if known)
  • _help (boolean) - list available options in JSON

add_ticket_document(self, ticket, url=None, name=None, base64=None, comment=None, content=None, _help=None)

source code 

Add a document to an existing ticket if the authenticated user can edit it.

Returns a JSON serialized object of the ticket if the call succeeds.

Special Note base64 and url are mutually exclusive.

Parameters:
  • ticket (integer) - ID of the ticket
  • url (string) - url of the document to be uploaded
  • name (string) - name of the document
  • base64 (base64 encoded string) - content of the document in base64 encoded string
  • comment (string) - depracated, use content instead
  • content (string) - if present, also add a followup, if the documentation upload succeeded, for additional options, see GLPI.add_ticket_followup
  • _help (boolean) - get usage information

add_ticket_followup(self, ticket, content, source=None, private=None, reopen=None, close=None, _help=None)

source code 

Add a followup to an existing ticket if the authenticated user can edit it.

Returns a hashtable of the modified ticket object (See GLPI.get_ticket).

Parameters:
  • ticket (integer) - ID of the ticket
  • content (string) - required content of the new followup
  • source (string) - name of the RequestType (created if needed), defaults to 'WebServices'
  • private (boolean) - mark followup as private, defaults to 0
  • reopen (boolean) - set ticket to working state (deny solution for 'solved' ticket or answer for 'waiting' ticket)
  • close (boolean) - close a 'solved' ticket
  • _help (boolean) - get usage information
